# Current State of `jedisherpa/wizardjoeavatar`

**Plain-English description.** WizardJoeAvatar builds and verifies a live animated Wizard Joe character for ASCILINE-style broadcast and chat surfaces. Instead of video or static images, the system generates colored character cell grids on the server, encodes them, streams over WebSocket, and draws them as square tiles in the browser. The original goal requires a procedural wizard that can stand, blink, speak, walk in depth and eight directions, follow paths, perform gestures, recover from reconnects, and remain stable in long-running broadcasts (`CODEX_GOAL.md:00007-00010`; `docs/00-goal-and-visual-contract.md:00005-00024`).

**Current architecture.** The Rust engine crate `rust/wizard_avatar_engine` is a library plus binaries declared in `rust/wizard_avatar_engine/Cargo.toml:00007-00042`. Binaries include `wizard-avatar-server`, `wizard-avatar-demo`, `wizard-avatar-snapshots`, `wizard-avatar-performance`, `wizard-avatar-evidence`, `wizard-avatar-soak`, `wizard-avatar-pose-catalog`, and `wizard-avatar-pose-evidence`. Dependencies are Axum, Tokio, tokio-tungstenite, serde, flate2, sha2 (`rust/wizard_avatar_engine/Cargo.toml:00043-00058`). The offline pose compiler is `rust/wizard_avatar_pose_tool` with binaries declared in `rust/wizard_avatar_pose_tool/Cargo.toml:00019-00029`. A standalone rchat validator exists in `tools/rchat/Cargo.toml:00001-00020`. Python package `wizardjoeavatar` depends on FastAPI, Uvicorn, Pillow (`pyproject.toml:00001-00010`; `requirements.txt:00001-00003`).

Major modules under `rust/wizard_avatar_engine/src/` cover animation, capability manifests, chat events/inbox/policy/performance, codec, commands, controller, evidence, frame source, hub, motion catalog/director/graph, pathing, performance score, pose assets/clips/program/playback, projection, renderer, runtime, server, state, and state regions. Browser clients live under `web/avatar/` and `rust/wizard_avatar_engine/web/`.
