## Confirmed Phase 0 Findings

- Python remains the performance authority; Prism remains the authority for
  governed turn state, approved language, TTS production, and audible timing.
- `WizardFrameHub` remains the single semantic writer. Character Director must
  feed the existing `CompiledScoreLoader` -> `PerformanceScheduler` ->
  `PerformanceApplication` path.
- Media Session V1 is preserved for content-free media state. Governed
  conversational direction requires a separate versioned message family over
  the existing authenticated relay and acknowledgement pattern.
- The authoritative media clock is Prism playback state. Frame-by-frame model
  calls and independent animation clocks are prohibited.
- Current rendering mutates state after hashing, replay and source-watermark
  collections are unbounded, browser message buffering is unbounded, and heavy
  frame production runs on the sole event-loop thread.
- Live presentation sampled about 10 FPS against a 24 FPS target. Professional
  animation authority cannot be enabled until the presentation budget passes.
- Diagnostic-only poses are currently exposed by normal Play, Repeat, and pose
  controls. Production performance must resolve only independently admitted
  capabilities.
- Prism currently lacks a backend-issued content approval envelope binding text,
  speech, and animation sinks. Unapproved content must fail closed before
  conversational performance is enabled.
