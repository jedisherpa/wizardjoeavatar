# Mira Solen Runtime Integration

Mira is registered as `mira-solen-v1` through the shared character registry. The browser selector, character-scoped REST routes, WebSocket stream, movement controller, projection, adaptive encoder, diagnostics, and reconnect behavior use the same production architecture as the existing characters while state and encoder history remain isolated per character.

Runtime assets:

- `mira_solen_pose_cells.json`: 108 worksheet-derived graphs from sheets 02–09. Full-body graphs are pose-capable; the 16 hand/prop feature graphs remain audited donor data.
- `mira_solen_pixel_graphs.json`: the 16 isolated identity graphs from approved identity sheet v3. Together these files are the exact 124-graph production set.
- `mira_solen_extraction_audit.json`: a 124-item pre-animation background, floor, contact-shadow, bounds, source-hash, node-count, and graph-hash audit.
- `mira_solen_animation_graph.json`: Mira-specific idle, locomotion, artwork reveal, critique invitation, dignity protection, creative reframing, student-voice, blue-flame, interaction, and turnaround clips.
- `mira_solen_animation_matrix.json`: supported ground, expression, speech, interaction, critique, artwork, dignity, and student-voice vocabulary.
- `mira_solen_character_manifest.json`: identity, attachment, derivation, and content-hash lineage.
- `mira_solen_runtime_profile.json`: data-driven state-to-pose mappings.
- `mira_solen_character_package.json`: loader and registry package.

Package loading recomputes the audit hash for every graph before the animation profile and controller are constructed. Runtime rendering reads transparent colored nodes from JSON and paints them into the ASCILINE framebuffer. It does not load a worksheet, contact sheet, PNG, SVG, data URI, or flattened sprite. Missing nodes remain transparent.

Regenerate with:

```bash
uv run python tools/generate_voxel_persona_character.py \
  assets/reference/personas/mira-solen/generation-profile.json
```

Append `--check` for the deterministic no-diff gate. Review-only contact evidence is generated by `tools/render_direct_cell_contact_evidence.py`; those PNGs are not package assets.
